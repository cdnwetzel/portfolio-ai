# Case Study: SOC 2 Type II Compliance — Multi-Tenant MSP

**Role:** Chris Wetzel — designed, executed, and managed this SOC 2 Type II program, and served as the primary technical point of contact with the audit firm.

**Organization:** Multi-tenant MSP (managed service provider)
**Framework:** SOC 2 Type II, rolling 90-day evaluation window
**Platform:** Secureframe (continuous compliance monitoring)
**Outcome:** Institutionalized continuous, auditable security operations across the MSP ecosystem — without disrupting developer velocity or daily engineering output

---

## The Problem

The MSP needed to move from standard engineering best practices to a fully compliant SOC 2 Type II framework — rapidly, inside a rolling 90-day evaluation window. SOC 2 Type II is not a one-time snapshot: auditors sample a continuous window of evidence, so each control has to actually operate, every day, across the whole period. The challenge was building that continuous, auditable security posture across a multi-tenant MSP — spanning on-premises and cloud environments — without slowing down the engineering teams whose daily output is the business.

---

## The Platform: Continuous Compliance Monitoring

Standardized the program on **Secureframe** for continuous compliance monitoring, and integrated **all core on-premises and cloud environments — including the M365 infrastructure** — into it. This built **automated evidence streams**: instead of scrambling to collect proof at audit time, evidence was generated continuously from the live environments, keeping the rolling 90-day window clean by default.

---

## Policies & Governance

Authored and implemented **over 20 formal corporate information security policies**, and **closed existing governance gaps** — turning informal "this is how we do it" practices into documented, enforceable, auditable controls. That is the foundation auditors actually test against: the policy, the evidence it is being followed, and proof it holds every day.

---

## Interfacing With the Audit Firm

Served as the **primary technical point of contact, interfacing directly with the audit firm** — translating between the technical environment and the auditors' requirements, and owning the evidence and control narrative end to end.

---

## The Constraint: No Disruption to Engineering

The hard requirement was institutionalizing continuous, auditable security operations **without disrupting developer velocity or daily engineering output**. Compliance had to be built into the existing environments and workflows — through automated evidence collection and integrated monitoring — rather than bolted on as friction.

---

## Outcome

The program institutionalized **continuous, auditable security operations across the multi-tenant MSP ecosystem** — a repeatable, evidence-backed SOC 2 Type II posture that holds across the rolling window, not a one-time audit scramble.

---

## Why It Worked

- **Continuous, not point-in-time** — automated evidence streams keep the rolling 90-day window clean every day, which is exactly what SOC 2 Type II tests.
- **Integrate, don't bolt on** — wiring Secureframe into the real on-prem / cloud / M365 environments meant compliance did not slow engineering.
- **Policy + evidence + proof** — 20+ formal policies plus continuous evidence gave the auditors the three things they need: the rule, that it is followed, and proof over time.
